# main

## Purpose

Binary entry point, system bootstrap, and trait-exists probe. Includes kernel modules via build.rs-generated `kernel_modules.rs`, provides `bootstrap()` to initialize the trait runtime (registry + dylibs + globals + dispatcher), `trait_exists()` to probe for trait definitions without bootstrapping, and delegates CLI execution to `cli::run()`.

## Exports

* `bootstrap(config: &Config) -> Result<Dispatcher>` — initializes registry, dylibs, globals, returns dispatcher
* `trait_exists(config: &Config, trait_path: &str) -> bool` — lightweight path probe for trait TOML existence

---

## Kernel module inclusion

All kernel modules (types, config, globals, registry, dispatcher, dylib_loader, cli, serve) are declared via `include!(concat!(env!("OUT_DIR"), "/kernel_modules.rs"))` — a file auto-generated by build.rs that emits `#[path = "..."] pub mod name;` for each kernel trait discovered in `traits/kernel/`. The `main` module itself is skipped (it IS the crate root).

---

## bootstrap

### Purpose

Initialize the full trait runtime: load registry from traits directory, initialize globals, load dylibs, and create a Dispatcher. Used by both CLI dispatch and HTTP server startup.

### Dependencies

* `registry::Registry` — loads .trait.toml files
* `globals::init()` — initializes global state
* `dylib_loader::DylibLoader` — loads .dylib plugins
* `dispatcher::Dispatcher` — executes traits

---

## trait_exists

### Purpose

Lightweight probe: checks if a trait definition TOML exists on disk without bootstrapping the runtime. Used by CLI to disambiguate `sys.*` vs `kernel.*` trait paths.

---

## main

### Purpose

Async entry point. Delegates to `cli::run()` which handles all clap parsing, config loading, and dispatch.

### Dependencies

* `cli::run()` — all CLI logic lives in `traits/kernel/cli/cli.rs`

1. Init tracing subscriber with stderr output
2. Load config from "traits.toml"
3. Parse CLI args
4. Match subcommand:
   - `Call { path, args }`: delegate to `cli::call_trait`
   - `External(args)`: prepend `sys.` to first arg, delegate to `cli::call_trait` with rest
   - `None` (no subcommand): check `TRAITS_PORT` env override, dispatch to `sys.serve`

### Edge Cases

* `TRAITS_PORT` env var overrides config port for default serve
* No subcommand defaults to serve with config port
* Config file missing or malformed returns error
* Any unknown subcommand (e.g. `traits list`) routes to `sys.list`

### Example

```bash
traits serve              # → sys.serve (via external_subcommand)
traits call sys.checksum hash hello
traits list               # → sys.list (via external_subcommand)
traits test_runner '*'    # → sys.test_runner (via external_subcommand)
traits                    # defaults to sys.serve with config port
```

---

## Internal Structure

Thin routing layer. Kernel modules are included from `traits/kernel/` via build.rs code generation. `bootstrap()` and `trait_exists()` live here at the crate root so they can use bare module paths. All CLI-specific work (clap parsing, arg formatting, stdin injection) lives in the `cli` module.

## Symlink

`src/` is a symlink to `traits/kernel/main/` — Cargo resolves `[[bin]] path = "src/main.rs"` through the symlink so the entry point lives alongside all other kernel traits.

## Notes

* `#[tokio::main]` makes this an async runtime entry point
* `trailing_var_arg = true, allow_hyphen_values = true` on Call args allows passing flags like `--verbose` as trait arguments
* `#[command(external_subcommand)]` catches all non-`call` subcommands and routes them as `sys.<name>`
